# 노력 기록 (Effort Tracker)

조용히 쌓이는 노력 — 즐겁게 한 노력도, 하기 싫었지만 해낸 노력도 — 을 기록하고
따뜻한 회고체 통계로 읽어주는 **모바일 우선 PWA**입니다. 서버·계정·결제 없이
100% 클라이언트(로컬 저장)로 동작합니다.

- React + Vite + TypeScript / 데이터는 `localStorage`
- 오프라인 동작(Service Worker) · 홈 화면 설치(PWA)
- GitHub Pages 정적 배포 (hash 라우팅으로 새로고침 404 없음)

## 화면

- **입력**: 날짜 네비(과거 소급 입력), 누적 카드(점수·시간·항목수·즐겁게/버텨냄 막대·연속 기록·**하루 목표 진행**), 항목 리스트, 하단 컴포저(내용·시간 스테퍼·저항도 슬라이더·카테고리)
- **통계(메인)**: 일/주/월/연 토글, **카테고리 필터**, 요약(증감 포함), **목표 달성 일수·페이스 예측**, **자동 인사이트 피드**, 한 줄 회고, 노력 흐름(누적 막대 + 이동평균), 즐겁게 vs 버텨냄 도넛, 카테고리 프리뷰
- **저항 분석**: 평균 저항도 변화, 회복 신호, 6개월 변화, 요일별 저항도, **저항도 분포 히스토그램**, **평일 vs 주말**, 요일×시간대 히트맵
- **카테고리 분석**: 누적 노력량, 성장 점수 랭킹, 노력 사분면(시간×저항)
- **노력 달력**: GitHub 잔디형 1년 히트맵(**칸 탭 → 그날 입력**, 목표 달성일 테두리) + 전체 누적
- **월간 리포트**: 영수증 감성 요약 + 노력 스타일 분류 + 텍스트·**이미지(PNG)** 저장/공유
- **최고기록 아카이브**: 가장 많이 버틴 날 등 6개 하이라이트
- **설정**: 저항 계수, **하루 목표 점수**, 카테고리 관리, 화면 테마, **전체 기록 누적**, 데이터 백업(JSON)·**CSV 내보내기**, 데모 데이터 토글

## 점수 공식

```
항목 투입시간(h) = units * 0.25      (units = 15분 단위 개수)
항목 노력량      = 투입시간 * (1 + resistance * resistanceCoef)   // 기본 coef = 0.3
하루 노력량      = 그날 항목 노력량의 합
```

- 저항도 0~5 — 0 편하게 함 / 1 살짝 귀찮음 / 2 미루고 싶었음 / 3 하기 싫었음 / 4 진짜 버팀 / 5 나를 이김
- **즐겁게 한 노력** = 저항 0~2, **버텨낸 노력** = 저항 3~5
- **주(week)** 단위는 달력상의 월~일이 아니라 **오늘 기준 직전 7일(롤링)**입니다. 일/월/연도 같은 방식으로 기준일(anchor)에 맞춰 계산합니다.

## 개발

```bash
npm install
npm run dev        # http://localhost:5173
npm run lint       # ESLint (flat config)
npm run typecheck  # tsc --noEmit
npm test           # Vitest 단위 테스트(통계 코어)
npm run build      # 아이콘 생성 → 타입체크 → dist/ 빌드
npm run preview    # 빌드 결과 미리보기
```

통계 계산 로직(`src/lib/score.ts`·`stats.ts`·`feed.ts`)은 순수 함수로 분리되어 있고
`src/lib/*.test.ts`(35개)에서 검증합니다. PR/푸시 시 `.github/workflows/ci.yml`이
lint·타입체크·테스트·빌드를 자동 실행합니다.

아이콘(PWA)은 `scripts/gen-icons.mjs`가 의존성 없이 PNG로 생성합니다
(`npm run build` 시 자동 실행).

## GitHub Pages 배포

1. **저장소 생성** 후 이 코드를 push 합니다.
2. **`base` 설정**: `vite.config.ts`의 `base`를 저장소 이름에 맞게 바꿉니다.
   - 예: `https://<user>.github.io/effort/` → `base: '/effort/'`
   - 현재 기본값은 `'/effort/'` 입니다. (`VITE_BASE` 환경변수로도 덮어쓸 수 있어요.)
3. **Pages 소스 설정**: 저장소 **Settings → Pages → Build and deployment → Source**를
   **GitHub Actions**로 선택합니다.
4. `main` 브랜치에 push 하면 `.github/workflows/deploy.yml`이 자동으로
   `npm ci && npm run build` 후 `dist/`를 Pages에 배포합니다.

## 모바일에서 설치하기

- **iPhone (Safari)**: 공유 버튼 → **‘홈 화면에 추가’**
- **Android (Chrome)**: 하단 설치 안내 또는 메뉴 → **‘앱 설치’**

설치하면 주소창 없이 앱처럼 열리고, 한 번 연 뒤에는 오프라인에서도 실행됩니다.

## 데이터 / 개인정보

모든 기록은 **기기의 브라우저 저장소(localStorage)**에만 보관됩니다. 서버로
전송되지 않으며 계정·로그인·결제가 없습니다. 데모 데이터와 실제 기록은 분리 저장되어
설정에서 언제든 켜고 끌 수 있습니다.
